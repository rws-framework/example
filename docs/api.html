API Documentation

Authentication Instructions

To use authenticated endpoints:

  1. First, obtain a token using either /api/login or /api/authorize
  2. Include the token in subsequent requests using the Authorization header:
    3. Authorization: Bearer your-token-here

Authentication Endpoints

POST /api/login

Authenticates a user with username and password, returns a JWT token.

Example Request:

curl -X POST http://your-api/api/login \ -H "Content-Type: application/json" \ -d '{ "username": "your-username", "passwd": "your-password" }'

Example Response:

{ "success": true, "data": { "user": { "id": "user-id", "username": "your-username" }, "token": "eyJhbGciOiJIUzI1NiIs..." } }

POST /api/authorize

Authenticates using an API key, returns a JWT token.

Example Request:

curl -X POST http://your-api/api/authorize \ -H "Content-Type: application/json" \ -d '{ "apikey": "re-12345678-uuid-format" }'

Example Response:

{ "success": true, "data": { "user": { "id": "user-id", "username": "your-username" }, "token": "eyJhbGciOiJIUzI1NiIs..." } }

User Management

GET /api/users/

Lists all users in the system. Requires authentication.

Example Request:

curl -X GET http://your-api/api/users \ -H "Authorization: Bearer eyJhbGciOiJIUzI1NiIs..."

Example Response:

{ "data": [ { "id": "1", "username": "user1", "active": true }, { "id": "2", "username": "user2", "active": true } ] }

POST /api/users/create

Creates a new user account. Requires authentication.

Example Request:

curl -X POST http://your-api/api/users/create \ -H "Authorization: Bearer eyJhbGciOiJIUzI1NiIs..." \ -H "Content-Type: application/json" \ -d '{ "username": "newuser", "passwd": "password123", "r_passwd": "password123" }'

Example Response:

{ "success": true, "data": { "id": "new-user-id", "username": "newuser", "active": true } }

GET /api/users/createkey/:id

Generates a new API key for a user. Requires authentication.

Example Request:

curl -X GET http://your-api/api/users/createkey/user-id \ -H "Authorization: Bearer eyJhbGciOiJIUzI1NiIs..."

Example Response:

{ "success": true, "data": { "id": "key-id", "keyval": "re-12345678-uuid-format" } }

Error Responses

Authentication Error:

{ "success": false, "data": null, "error": "Invalid credentials" }

Validation Error:

{ "success": false, "data": "Passwords do not match" }

Security Notes

  • All tokens expire after a certain period (check with system administrator for exact duration)
  • API keys are permanent until deleted and should be kept secure
  • Always use HTTPS for requests to ensure data security
